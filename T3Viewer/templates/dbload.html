Overview
This form allows you to load the database with the telemetry data from S3.
The data is uploaded to a set of temporary tables prefixed with the prefix you specify.
If the tables don't exist, they will be created.
If the tables exist, they will be loaded with the data for the specified period.
Keep in mind that RedhShift doesn't enforce primary key or uniqueness constraints.
If you run the same upload twice, the data will be uploaded twice.
The period can specified as a range of days.
You can choose to upload data for all event classes or for a specific event class.
The current form only allows you to upload a max of a month of data. This limitation is to restrict inadvertent data load.
